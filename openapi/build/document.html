Y-Sweet Document-level API (0.5.0)

Download OpenAPI specification:Download

Y-Sweet has two APIs:

  1. A document management API, which allows creating and updating documents.
  2. A single document API (this one), which allows connecting to and updating a single document.

Typically, you will first use the document management API to get an auth token for a specific document (via /doc/{docId}/auth), which will return a baseUrl. That baseUrl will become the base for the single document API described here. All API calls are scoped to the document ID provided in the auth request path.

Get As Yjs Update

Retrieves the document in Yjs update format, which provide a compressed binary representation of a set of changes to a Yjs document.

Updates can be applied to a Yjs document as follows:

import * as Y from 'yjs'

let doc = new Y.Doc()
doc.transact(() => {
  Y.applyUpdate(doc, update)
})
Authorizations:
bearerAuth

Responses

/as-update

Update With Yjs Update

Writes to a document with the provided Yjs update, which provide a compressed binary representation of a set of changes to a Yjs document.

Updates can be obtained from a Yjs document as follows:

import * as Y from 'yjs'

// doc is a Y.Doc
let update = Y.encodeStateAsUpdate(doc)

Note that this will encode the entire document state as an update, but you can also use this endpoint for incremental updates. See Yjs's Document Updates documentation for more information.

Authorizations:
bearerAuth
Request Body schema: application/octet-stream
required
string <binary>

Responses

Connect to a document over WebSocket.

Establishes a WebSocket connection for real-time synchronization with the y-websocket protocol.

path Parameters
docId
required
string

Document ID. This must match the docId returned from the /doc/{docId}/auth endpoint in the request that provided the current baseUrl.

query Parameters
token
string

Client token for authentication. If the document manager returned a token in the auth response, it must be provided here.

Responses

/y-websocket/{docId}

  • 401
